Abstract Classes and Interfaces

Pre Java 8
  • Interfaces - no methods or state implemented
  • Abstract Classes - some methods or state implemented
  • Concrete Classes - all methods and all state implemented
Java 8
  • Interfaces can now contain default method implementations
    • interfaces are supposed to be immutable
    • breaks the idea of pure contract ...
    • but allows changes to interface that don't break existing code
    • needed because of the changes required by collection classes to support lambdas and streams
  • Multiple Inheritance now possible
    • can inherit from multiple interfaces
    • can inherit from only one abstract class
Interfaces
  • can contain abstract methods
  • can contain default methods that have an implementation
  • methods are inherently public
  • fields are inherently public, static, and final
Abstract Classes
  • state can be non-static and non-final
  • methods can be public, private, or protected
